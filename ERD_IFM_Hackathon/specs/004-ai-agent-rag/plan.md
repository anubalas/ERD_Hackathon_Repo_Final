# Implementation Plan: AI Agent — GMP Remediation with RAG

**Branch**: `004-ai-agent-rag` | **Date**: 2026-06-09 | **Spec**: `specs/004-ai-agent-rag/spec.md`

**Input**: Feature specification from `specs/004-ai-agent-rag/spec.md`

---

## Summary

Build a GMP remediation AI agent that polls the SQLite `alerts` table for unprocessed anomaly
alerts, retrieves relevant SOP guidance from a ChromaDB vector store via semantic search
(RAG), calls Claude `claude-sonnet-4-6` with prompt caching enabled, extracts a structured
citation and confidence score from the response, and persists an `AgentRun` audit record.
Includes: an offline RAG ingestion script for four sample SOP text files, the `AgentRun` ORM
extension, and a new "AI Recommendations" tab in the Streamlit operator dashboard.

---

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**:
- `anthropic` — Direct SDK for Claude calls with `cache_control: ephemeral` per-block support
- `chromadb` — Vector store with built-in `DefaultEmbeddingFunction` (all-MiniLM-L6-v2, local)
- `SQLAlchemy 2.0` (sync) — `AgentRun` DB writes + Streamlit dashboard queries
- `python-dotenv` — ANTHROPIC_API_KEY, SQLITE_DB_PATH, CHROMA_PERSIST_DIR from .env
- `langchain` — `PromptTemplate` for system prompt management only (no LLM abstraction)
- `streamlit` — Extend existing dashboard with AI Recommendations tab

**Storage**:
- SQLite (`ifm_audit.db`) — `agent_runs` table (append-only)
- ChromaDB (`./chroma_db`) — `gmp_docs` collection, local disk (Constitution Principle V)

**Testing**: pytest + unittest.mock (no asyncio needed — agent polling loop is synchronous)

**Target Platform**: Local development (Windows 11 / factory Linux server)

**Project Type**: Background service (polling agent process) + offline CLI tool (ingestion script)

**Performance Goals**:
- Each alert processed (RAG + Claude + AgentRun written) within 10 seconds of pickup (SC-001)
- RAG ingestion of 4 sample docs completes in under 30 seconds (SC-005)
- Dashboard AI Recommendations tab loads in under 2 seconds (SC-004)

**Constraints**:
- `cache_control: ephemeral` on both system prompt and SOP context blocks (Constitution Principle VII)
- No external embedding API — ChromaDB default MiniLM runs locally (Constitution Principle V)
- AgentRun table is append-only — no UPDATE or DELETE permitted (Constitution Principle II)
- ChromaDB collection never modified during normal alert processing (Constitution Principle IV analog)
- Model is always `claude-sonnet-4-6` — no fallback to Haiku for GMP decisions

**Scale/Scope**: Demo scale — up to ~10 alerts per demo session. Serial processing, no concurrency.

## Constitution Check

| Principle | Requirement | Plan Compliance |
|-----------|-------------|-----------------|
| I — Zero Silent Failures | Claude API errors logged with alert_id; alert remains unprocessed for retry | `agent.py`: `except anthropic.APIError` → log + continue; alert stays in unprocessed set |
| II — Immutable Audit Log | AgentRun append-only; no UPDATE/DELETE | `crud.py` adds `create_agent_run()` only; no update/delete functions |
| III — AI Citation Mandate | Every response validated for citation; failures set requires_human_review=True | `extract_json_block()` → CITATION_VIOLATION log on failure |
| V — On-Premise Data Sovereignty | ChromaDB + SQLite on local disk; no external embedding API | DefaultEmbeddingFunction runs locally; no OpenAI/Cohere embedding calls |
| VI — Human-in-the-Loop | requires_human_review=True when confidence < 0.7 or citation absent | FR-009 enforced in `agent.py` before `create_agent_run()` |
| VII — Prompt Caching | `cache_control: ephemeral` on both system prompt and SOP context | Three-block message structure with explicit cache_control on blocks 1 and 2 |

**Gate result**: PASS — no violations.

## Implementation Phases

### Phase 1: GMP Document Knowledge Base (US1 — P1)

**Goal**: ChromaDB ingestion pipeline working end-to-end. Query returns top-3 chunks with
source metadata. Idempotent re-runs don't duplicate chunks.

- Create `docs/gmp/` directory with four sample `.txt` SOP files (FR-013):
  - `boiler_sop.txt` — steam boiler CCPs: temperature 150–170°C, pressure 4–7 bar, §1–§4
  - `pasteurizer_sop.txt` — pasteuriser CCPs: temperature ≥72°C, pH 6.5–7.0, flow ≥80 L/h
  - `dryer_sop.txt` — spray dryer CCPs: temperature 100–130°C, humidity ≤40%
  - `haccp_general.txt` — HACCP deviation response, corrective actions, verification steps
- Create `src/rag/__init__.py` (empty)
- Implement `src/rag/chroma_store.py`:
  - `init_collection(client, name)` → delete-then-create for idempotency (Decision 5)
  - `query_collection(collection, query_string, n_results=3)` → returns `List[GmpChunk]`
- Implement `src/rag/ingest.py`:
  - `chunk_document(text, chunk_size=500, overlap=50)` → `List[str]` (Decision 4)
  - `ingest_docs(docs_dir, chroma_dir, collection_name)` → reads all `.txt`, chunks, embeds, stores
  - `__main__` block with argparse (`--docs-dir`, `--chroma-dir`, `--collection`)
  - Exit with error message if no `.txt` files found (AC-4 from US1)

### Phase 2: DB Schema Extension (AgentRun — prerequisite for US2)

**Goal**: `AgentRun` ORM model and `create_agent_run()` / `get_unprocessed_alerts()` in place.

- Add `AgentRun` ORM model to `src/db/models.py`:
  - Fields: `id`, `alert_id`, `recommendation`, `citation`, `confidence_score`,
    `requires_human_review`, `model_name`, `raw_response`, `created_at`
  - No UPDATE or DELETE functions (Constitution Principle II, Decision 8)
- Add to `src/db/crud.py`:
  - `create_agent_run(session, ...)` — INSERT-only, synchronous (Decision 9)
  - `get_unprocessed_alerts(session)` — returns `List[Alert]` with no AgentRun record
- Ensure `init_db()` creates the `agent_runs` table

### Phase 3: AI Agent Polling Loop (US2 — P2)

**Goal**: Agent polls alerts, runs RAG, calls Claude with caching, extracts citation, saves
AgentRun. Handles Claude API failure without losing the alert.

- Create `src/agent/__init__.py` (empty)
- Implement `src/agent/prompts.py`:
  - `GMP_SYSTEM_PROMPT` — role: GMP compliance agent for infant food manufacturing
  - Instructs Claude to: always cite SOP source + §clause, state confidence as float,
    include "REQUIRES HUMAN REVIEW" if uncertain, output JSON block at end
  - JSON block format: `{"citation": "...", "confidence": 0.85, "requires_human_review": false}`
- Implement `src/agent/agent.py`:
  - `extract_json_block(text)` — regex extraction, returns dict or None (Decision 6)
  - `build_query(alert)` — constructs GmpQuery string from alert fields
  - `process_alert(alert, collection, client, session)` — full pipeline for one alert:
    1. RAG query → top-3 chunks
    2. Format SOP context block
    3. Call `client.messages.create()` with three-block cached structure (Decision 7)
    4. Extract JSON block; fallback on failure
    5. Clamp confidence to [0.0, 1.0]; apply AGENT_HUMAN_REVIEW_THRESHOLD
    6. `create_agent_run(session, ...)`
  - `polling_loop(session, collection, client, interval)` — `while True` + sleep
  - `__main__` block: startup checks → polling loop

### Phase 4: Streamlit Dashboard Extension (US3 — P3)

**Goal**: Third "AI Recommendations" tab displays AgentRun records colour-coded by confidence.

- Extend `src/dashboard/app.py`:
  - Add third tab `"🤖 AI Recommendations"` alongside existing "Live Feed" and "Alerts" tabs
  - Query: JOIN `agent_runs` with `alerts`, ORDER BY `created_at DESC`, LIMIT 20
  - Colour-code rows using `st.markdown` with inline CSS or `st.dataframe` style:
    - Green: `confidence_score >= 0.7`
    - Amber: `0.4 <= confidence_score < 0.7` or `requires_human_review = True`
    - Red: `confidence_score < 0.4` or `citation = ""`
  - Show "REQUIRES HUMAN REVIEW" badge for flagged records
  - Empty state message when no AgentRun records exist (AC-4 from US3)
  - Uses existing sync `SessionLocal` (Decision 9 — no new session infrastructure)

### Phase 5: Unit Tests

**Goal**: Full unit test coverage per SC-006.

- `tests/unit/test_rag_ingest.py`:
  - `chunk_document()` produces correct chunk count and overlap
  - `query_collection()` returns top-3 results with `source` metadata
  - Idempotent re-ingest (mock collection delete+recreate)
  - Empty docs directory → error raised
- `tests/unit/test_citation_extraction.py`:
  - `extract_json_block()` returns dict on valid JSON block
  - `extract_json_block()` returns None on malformed or absent block
  - Confidence clamped to 0.0 when negative, 1.0 when > 1
  - `requires_human_review` forced True when confidence < threshold
- `tests/unit/test_agent_run_crud.py`:
  - `create_agent_run()` writes correct fields to in-memory SQLite
  - `get_unprocessed_alerts()` skips alerts with existing AgentRun records
  - Append-only contract: assert no `update_agent_run`/`delete_agent_run` in `crud.py`

## Key Technical Decisions

| Decision | Choice | Reference |
|----------|--------|-----------|
| LLM client | Direct Anthropic SDK (not LangChain) | research.md §1 |
| Embedding | ChromaDB DefaultEmbeddingFunction (MiniLM, local) | research.md §2 |
| Alert polling | `while True` + 10s sleep | research.md §3 |
| Document chunking | Fixed 500-char chunks, 50-char overlap | research.md §4 |
| Collection idempotency | Delete + recreate on each ingestion run | research.md §5 |
| Structured output | JSON block at end of free-text response | research.md §6 |
| Prompt caching | Three-block structure: GMP role (cached) + SOP context (cached) + alert (user) | research.md §7 |
| AgentRun append-only | INSERT-only crud; unprocessed = no AgentRun JOIN | research.md §8 |
| Dashboard session | Sync SQLAlchemy SessionLocal (same as existing dashboard) | research.md §9 |
