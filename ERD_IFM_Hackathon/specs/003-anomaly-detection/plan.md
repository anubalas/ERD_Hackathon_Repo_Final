# Implementation Plan: Anomaly Detection

**Branch**: `003-anomaly-detection` | **Date**: 2026-06-09 | **Spec**: `specs/003-anomaly-detection/spec.md`

**Input**: Feature specification from `specs/003-anomaly-detection/spec.md`

---

## Summary

Build a scikit-learn IsolationForest anomaly detection service that subscribes to the Redis
`telemetry` pub/sub channel, scores each incoming reading using per-device-type models trained
on `data/telemetry_baseline.csv`, and writes `Alert` records (ANOMALY or PIPELINE_ERROR) to
the SQLite audit database. Includes an offline training script and an async subscriber process.

---

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**:
- `scikit-learn` ‚Äî IsolationForest model training and scoring
- `joblib` ‚Äî Model serialisation/deserialisation (bundled with scikit-learn)
- `redis[asyncio]` ‚Äî Async Redis pub/sub subscriber
- `SQLAlchemy 2.0` (async) ‚Äî Alert DB writes (reuses existing `src/db/` setup)
- `pandas` ‚Äî Baseline CSV loading during training only
- `python-dotenv` ‚Äî ANOMALY_THRESHOLD, REDIS_URL, SQLITE_DB_PATH from .env

**Storage**: SQLite (existing `ifm_audit.db`) + local `.pkl` files in `src/detection/models/`

**Testing**: pytest + pytest-asyncio (asyncio_mode = auto) + unittest.mock

**Target Platform**: Local development (Windows 11 / factory Linux server)

**Project Type**: Background service (subscriber process) + offline CLI tool (training script)

**Performance Goals**: Score each telemetry message within 100 ms (SC-001); training completes
in under 60 seconds for up to 10,000 rows per device type (SC-005).

**Constraints**:
- Model is read-only at runtime (Constitution Principle IV)
- All storage on local disk (Constitution Principle V)
- Async subscriber loop ‚Äî no blocking Redis calls (Constitution Principle VII)
- `try/except Exception: pass` banned on any sensor data path (Constitution Principle I)

**Scale/Scope**: Up to 3 device types √ó 1 reading every 5 seconds = ~36 readings/minute.
Single-threaded async loop is sufficient.

---

## Constitution Check

| Principle | Requirement | Plan Compliance |
|-----------|-------------|-----------------|
| I ‚Äî Zero Silent Failures | Scoring errors write PIPELINE_ERROR alert; never swallowed | `anomaly.py`: `except Exception as exc` ‚Üí write alert, continue |
| II ‚Äî Immutable Audit Log | Alert table append-only; no UPDATE/DELETE | `crud.py` has `create_alert()` only; no update/delete functions defined |
| IV ‚Äî Anomaly Model Immutability | Model loaded once via `joblib.load()`; no `fit()` at runtime | `subscriber.py` loads models at startup; no `fit` call reachable from subscriber |
| V ‚Äî On-Premise Data Sovereignty | PKL files and SQLite on local disk; no cloud egress | No remote storage; `.pkl` in `src/detection/models/` |
| VII ‚Äî Safety-First / Async-first | Subscriber uses `redis.asyncio`; event loop not blocked | `subscriber.py` uses `async def` throughout |

**Gate result**: PASS ‚Äî no violations.

## Implementation Phases

### Phase 1: Setup & DB Schema Extension

**Goal**: Alert ORM model and `create_alert()` in place; database schema up to date.

- Add `Alert` SQLAlchemy ORM model to `src/db/models.py`
- Add `create_alert()` INSERT function to `src/db/crud.py` (no update/delete)
- Ensure `init_db()` creates the `alerts` table
- Create `src/detection/__init__.py` (empty)
- Create `src/detection/models/` directory

### Phase 2: Training Script

**Goal**: `python -m src.detection.anomaly --fit` produces `.pkl` files.

- Implement `AnomalyScorer` class in `src/detection/anomaly.py`:
  - `DEVICE_FEATURES` registry
  - `load_models(models_dir)` ‚Äî loads all device `.pkl` files; raises on missing
  - `score(device_type, payload_dict)` ‚Äî returns `(score: float, is_anomaly: bool)`
- Implement `train(data_path, output_dir, version)` function
- Implement `__main__` block with argparse (`--fit`, `--data-path`, `--output-dir`, `--version`)
- Create `data/telemetry_baseline.csv` with synthetic clean baseline rows (50+ per device type)

### Phase 3: Async Subscriber

**Goal**: `python -m src.detection.subscriber` subscribes, scores, writes alerts.

- Implement `src/detection/subscriber.py`:
  - Load `AnomalyScorer` at startup; `sys.exit(1)` if models missing
  - Connect to Redis via `redis.asyncio.from_url(REDIS_URL, protocol=2)`
  - Subscribe to `telemetry` channel
  - `process_message(raw, scorer, session)`:
    - Decode JSON ‚Üí dict
    - Extract device_type; skip unknown device types with WARNING log
    - Call `scorer.score(device_type, payload)`
    - If `is_anomaly`: `await create_alert(session, alert_type="ANOMALY", ...)`
    - On any exception: `await create_alert(session, alert_type="PIPELINE_ERROR", error_detail=str(exc), ...)` then continue
  - Reconnect loop with exponential backoff (1s ‚Üí 2s ‚Üí 4s ‚Üí ... ‚Üí 30s cap)
  - Graceful SIGINT/SIGTERM shutdown via `asyncio.Event`

### Phase 4: Unit Tests

**Goal**: Full unit test coverage per SC-006.

- `tests/unit/test_anomaly_scorer.py`:
  - Feature extraction per device type
  - Above-threshold (OK) and below-threshold (ANOMALY) scoring with mock model
  - Unknown device type ‚Üí skip (no exception)
  - Missing model file at load time ‚Üí raises on startup
- `tests/unit/test_subscriber.py`:
  - `process_message` with normal reading ‚Üí no alert written
  - `process_message` with anomalous reading ‚Üí ANOMALY alert written
  - `process_message` with malformed JSON ‚Üí PIPELINE_ERROR alert written
  - `process_message` with scoring exception ‚Üí PIPELINE_ERROR alert written
- `tests/unit/test_alert_crud.py`:
  - `create_alert()` writes correct fields to in-memory SQLite
  - Append-only contract: assert no `update_alert`/`delete_alert` in `crud.py`
