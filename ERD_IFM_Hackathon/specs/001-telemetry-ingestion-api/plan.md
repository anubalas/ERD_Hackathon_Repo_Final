# Implementation Plan: Telemetry Ingestion API

**Branch**: `001-telemetry-ingestion-api` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-telemetry-ingestion-api/spec.md`

---

## Summary

Implement an async `POST /telemetry` endpoint that receives CCP sensor readings from factory
devices (boiler, pasteurizer, dryer), validates structure via Pydantic and CCP operating
ranges in the route handler, writes every reading (accepted and rejected) to the SQLite audit
log as a single append-only INSERT, and publishes accepted readings to a Redis pub/sub channel
for downstream anomaly detection.

Redis failures are caught, logged, and reflected in the response (`stream_published: false`)
without losing the persisted record. Database unavailability returns HTTP 503. CCP violations
return HTTP 422 with per-field detail. No exception is swallowed anywhere on the sensor
pipeline.

---

## Technical Context

**Language/Version**: Python 3.11

**Primary Dependencies**: FastAPI 0.111+, Pydantic v2, `redis[asyncio]` ≥ 4.2,
SQLAlchemy 2.0 (async), `aiosqlite`, pytest 8+, pytest-asyncio, httpx

**Storage**: SQLite via `aiosqlite` + SQLAlchemy async engine (local file);
Redis 7+ pub/sub on `localhost:6379`

**Testing**: pytest + pytest-asyncio; httpx `AsyncClient` for route tests;
`unittest.mock.AsyncMock` for DB and Redis isolation in unit tests

**Target Platform**: Local development server (Windows/Linux), factory on-premise network

**Project Type**: Async REST web service — single ingestion endpoint, v1 baseline

**Performance Goals**: ≤ 300 ms p95 response under 50 concurrent devices (spec SC-001)

**Constraints**:
- No Docker — Redis and the app run locally
- No authentication in v1
- No cloud storage — SQLite and Redis are on-premise only
- `async def` for all FastAPI routes and DB/Redis I/O
- No `try/except Exception: pass` on any code path that handles sensor data, audit writes,
  or Redis publishing

**Scale/Scope**: Single factory, ≤ 50 concurrent CCP sensor devices in v1

## Implementation Phases

### Phase 1 — DB Layer (Foundation)

**Files**: `src/db/models.py`, `src/db/database.py`

- `TelemetryLog` ORM model: all columns from data-model.md; no `update()`/`delete()`
  methods; `created_at` set server-side at INSERT time and never writable again
- Async SQLAlchemy engine: `create_async_engine("sqlite+aiosqlite:///${SQLITE_DB_PATH}")`
- `AsyncSession` factory: `async_sessionmaker(engine, expire_on_commit=False)`
- `init_db()`: creates all tables if absent; called in lifespan

### Phase 2 — Schemas and CCP Logic

**File**: `src/api/schemas.py`

- `DeviceType = Literal["boiler", "pasteurizer", "dryer"]` (case-insensitive normalised
  on input)
- `CCPRange(min: float, max: float)` dataclass
- `CCP_THRESHOLDS: dict[str, DeviceThresholds]` — see data-model.md for exact values
- `SensorReading` Pydantic model: structural validation only (types, required fields,
  non-negative values); CCP range checking is **not** a Pydantic validator — it runs in
  the route handler so the DB write of the REJECTED record can precede the 422 return
- `validate_ccp_ranges(reading: SensorReading) → list[CCPViolation]` — pure, no I/O
- `TelemetryResponse` model: `reading_id`, `status`, `server_received_at`,
  `stream_published`, `warnings: list[str]`
- `CCPViolation` model: `field`, `message`, `received`, `allowed_range`

### Phase 3 — CRUD

**File**: `src/db/crud.py`

- `async create_telemetry_log(session, *, reading, reading_id, server_received_at,
  status, rejection_reason, stream_published, stale_timestamp) → TelemetryLog`
- Single `session.add()` + `await session.commit()` — no other mutations
- SQLAlchemy exception propagates to caller; route handler catches for HTTP 503

### Phase 4 — Redis Streaming Client

**File**: `src/streaming/redis_client.py`

- `RedisClient` class wrapping `redis.asyncio.Redis.from_url(REDIS_URL)`
- `async publish_telemetry(reading_id, reading, server_received_at)`: serialises
  `TelemetryEvent` to JSON; calls `client.publish(channel, payload)`
- Raises `RedisPublishError(message, cause)` on failure — never suppresses
- `async get_redis_client() → AsyncGenerator[RedisClient, None]` FastAPI dependency

### Phase 5 — Route Handler and App Factory

**Files**: `src/api/routes/telemetry.py`, `src/api/main.py`

Route handler execution order (Principle II-compliant — single INSERT, correct
`stream_published` value at write time):

```
1. Pydantic validates structure          → automatic HTTP 422 if structural failure
                                           (no DB write — can't persist malformed data)
2. validate_ccp_ranges(reading)          → collect all violations in one pass
3. If violations exist:
     write TelemetryLog(REJECTED)        → if DB fails: log error, still return 422
     return HTTP 422 with CCPViolation list
4. Try Redis publish                     → stream_ok = True / False; log on failure
5. Write TelemetryLog(ACCEPTED,
       stream_published=stream_ok)       → single INSERT with correct value
   If DB fails → HTTP 503
6. Return TelemetryResponse(
       stream_published=stream_ok,
       warnings=[...] if not stream_ok)
```

`main.py`:
- `@asynccontextmanager lifespan`: `await init_db()` on startup; Redis client teardown
  on shutdown
- `app.include_router(telemetry_router, prefix="")`
- `@app.exception_handler(Exception)`: log full traceback; return HTTP 500 — never silent

### Phase 6 — Unit Tests

**Directory**: `tests/unit/`

| File | Scenarios covered |
|------|------------------|
| `test_schemas.py` | Valid reading each device type; missing required fields; wrong field types; negative measurements; unknown device_type; stale timestamp flag |
| `test_schemas.py` | CCP violations: each field per device, boundary values inclusive, multi-field simultaneous; `validate_ccp_ranges` returns empty list for valid readings |
| `test_crud.py` | Successful INSERT returns populated `TelemetryLog`; SQLAlchemy exception propagates (not swallowed); no `update_*`/`delete_*` callable on `crud` module |
| `test_telemetry_route.py` | Happy path (boiler, pasteurizer, dryer); CCP 422 with per-field detail; unknown device_type 422; Redis down → 200 + `stream_published: false` + warning; DB down → 503 |
